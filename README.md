# Byte — студия цифровых сервисов

Статический сайт студии «Byte».

## Публикация

- Сайт: https://ivanhhgdx.github.io/byte-studio-site/
- Репозиторий: https://github.com/Ivanhhgdx/byte-studio-site
- Статус: опубликован через GitHub Pages.
- Источник публикации: ветка `main`, корень репозитория (`/`).

После отправки изменений в ветку `main` GitHub Pages автоматически обновляет сайт. Обычно обновление занимает несколько минут.

## Где вносить изменения

Рабочая версия сайта находится именно в этом проекте:

`/Users/ivan/Documents/проект Студия цифровых сервисов`

Основные файлы:

- `index.html` — структура и тексты сайта;
- `style.css` — внешний вид и адаптивность;
- `script.js` — анимации и интерактивность;
- `assets/` — изображения и другие ресурсы.

Не нужно редактировать адрес GitHub Pages напрямую. Сначала измените файлы проекта, проверьте результат локально, затем отправьте изменения в GitHub.

## Как обновить опубликованный сайт

Перед началом новой работы получите актуальную версию:

```bash
cd "/Users/ivan/Documents/проект Студия цифровых сервисов"
git pull --rebase origin main
```

После внесения и проверки изменений:

```bash
git status
git add index.html style.css script.js assets/
git commit -m "Кратко описать изменение"
git push origin main
```

Добавляйте через `git add` только те файлы, которые действительно относятся к текущему изменению. Если менялись не все перечисленные файлы, уберите лишние пути из команды.

Пример:

```bash
git add index.html style.css
git commit -m "Обновить первый экран"
git push origin main
```

После `git push` проверьте опубликованную версию:

https://ivanhhgdx.github.io/byte-studio-site/

Если изменения ещё не появились, подождите 1–5 минут и обновите страницу без кеша.
